# ippo フロントエンド テスト方針

最終更新: 2026-06-28

---

## テスト構成

| レイヤー | ツール | 対象 |
|---|---|---|
| ユニット・結合 | vitest + React Testing Library + MSW | コンポーネント・hook・フォームバリデーション・API 連携 |
| E2E | Playwright | ユーザーフロー全体（実ブラウザ） |

- API 呼び出しは MSW（Mock Service Worker）でインターセプトし、結合テストではバックエンドを起動しない
- E2E は実際のバックエンド（またはシードした test DB）に対して実行する
- React Compiler 使用のため memo 化のテストは対象外。挙動（振る舞い）をテストする

---

## 結合テスト（vitest + RTL + MSW）

ユーザーから見た振る舞いをテストする。実装詳細（state 名など）はテストしない。

| 対象 | テストケース |
|---|---|
| LoginForm | 空欄 submit → zod バリデーションエラー表示 |
| | 正常入力 → API 呼出 → ログイン後画面へ遷移 |
| | API 401 → 「メールまたはパスワードが違います」表示 |
| RegisterForm | パスワード 8 文字未満 → エラー表示 |
| | 正常入力 → 登録成功 → ログイン状態に遷移 |
| 価値一覧 | `/api/values` モック → リスト描画 |
| | レスポンス空 → 空状態（empty state）表示 |
| 価値作成 | 空欄 submit → エラー |
| | 正常 submit → キャッシュ invalidate → リスト更新 |
| | API 400「この価値はすでに登録されています」→ UI にエラー表示 |
| 曝露記録作成 | anxiety_before に 101 入力 → zod エラー |
| | 価値未選択 → エラー |
| | 正常入力 → POST 送信 → 一覧へ反映 |
| 実施後追記 | done_at + anxiety_after 入力 → PATCH 送信 |
| ローディング | データ取得中 → skeleton 表示／送信中 → spinner 表示 |

## E2E テスト（Playwright）

主要なユーザーシナリオを通しで確認する。

| シナリオ | 確認内容 |
|---|---|
| ハッピーパス | 登録 → 価値追加 → 曝露記録作成 → 実施後追記（完全フロー） |
| セッション継続 | ログアウト → 再ログイン → データが保持されている |
| 認証ガード | 未ログインで `/values` アクセス → `/login` にリダイレクト |
| データ分離 | 別ユーザーの記録が見えないこと |
| 重複価値 | 同名価値を 2 回登録 → エラーメッセージ表示 |
| カレンダー（F6） | 日付をタップ → その日の記録のみ表示 |

## 実行コマンド

```bash
# 結合テスト
pnpm test            # vitest watch
pnpm test --run      # CI 用（1 回実行）

# E2E
pnpm test:e2e        # Playwright
```

lefthook の pre-push で結合テストを実行し、回帰を防ぐ。
