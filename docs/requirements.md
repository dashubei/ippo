# ippo 要件定義書（MVP・凍結）

最終更新: 2026-06-24
ステータス: **凍結（MVP スコープ確定）**

---

## 1. プロダクト概要
- **一言**：社交場面での不安に、価値に紐づけた小さな一歩（曝露）を記録・振り返るセルフヘルプ Web アプリ。
- **解決したい課題**：社交場面で不安を感じる人は、怖い行動を避け続けてしまいがち。曝露（少しずつ慣れる取り組み）を、自分の価値に結びつけて「続けやすく」する。
- **ターゲット**：社交場面で不安を感じる思春期以上の人（成人を含む）。
- **ポジショニング**：**非医療のセルフヘルプ／教育ツール**。治療・診断・寛解は標榜しない。

## 2. なぜバックエンドが必要か
本アプリはフロントだけでも記録の体裁は作れるが、次の価値を実現するためにサーバー側が必要：
1. **記録の永続化**：端末を変えても自分の記録を参照できる。
2. **アカウントごとの非公開保持**：記録は本人だけが見られる。
→ ゆえに **ユーザー認証 + ユーザー単位のデータ分離** が必須要件となる。

## 3. 設計の背景（心理学的な根拠）
- **曝露（エクスポージャー）**：不安な状況に段階的に取り組み、「慣れ（habituation）」を促す。
- **価値の明確化（ACT）**：「どんな人でありたいか」という価値を言語化し、行動をそこに結びつける。
- ippo は **「価値」を先に定義し、個々の曝露記録をその価値に紐づける** 構造を採用する（価値 → 行動の順）。これは「価値の言語化が、行動の自然な結果の動機づけ効果を高める（行動分析でいう augmenting）」という考え方に基づく。

## 4. 機能要件（MVP）
| # | 機能 | 内容 |
|---|---|---|
| F1 | アカウント | 登録・ログイン。ゲストログイン機能は設けず、**自己登録方式**（公開サイトから新規登録して試用してもらう）とする |
| F2 | 価値の登録 | ユーザーが自分の「価値」を登録・保持する |
| F3 | 曝露記録の作成 | 行動内容／実施前の不安(0–100)／紐づく価値／計画メモ |
| F4 | 実施後の追記 | 実施日時／実施後の不安(0–100)／実施後メモ |
| F5 | 一覧・詳細 | 自分の記録のみ閲覧（**他人の記録は見えない**＝データ分離） |
| F6 | カレンダー可視化 | 実施日を俯瞰。日をタップで当日の記録（前後の不安・価値・メモ）を表示 |

## 5. 非機能要件（MVP）
- **データ分離**：ユーザーは自分のデータのみアクセス可能（API 層で認可を担保）。
- **セキュリティ**：
  - パスワードはハッシュ化して保存。通信は HTTPS。秘密情報は環境変数で管理。
  - 主要な Web 脆弱性に対策：SQL インジェクション／XSS／CSRF／ブルートフォース。
  - 精神面に関わる**機微な個人データ**として丁寧に扱う（医療ではないが配慮する）。
- **可用性・コスト**：常時公開は無料枠で行い、固定費ゼロを目指す。
- **動作保証**：本番環境で確実に動作すること。

## 6. スコープ外（v2 以降に明記して見送る）
- 価値の明確化を解説するフルチュートリアル（モダン UI のオンボーディング）
- AI による曝露課題の提案（安全性・責任の線引きが必要なため、MVP では意図的に除外）
- 「慣れ」の折れ線グラフ／同一曝露の推移分析
- 支援者（カウンセラー等）との記録共有
